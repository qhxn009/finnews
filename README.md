## 含义与解析
指令主要由三条规则组成，旨在解决网页自动化操作中常见的稳定性问题。

1. 规则一：应对页面结构变化
 * 网站更新后，其 HTML 结构（DOM）可能会发生变化。如果程序尝试从页面提取信息（如抓取标题、正文）失败了，程序不应该崩溃或胡乱猜测，而是应该明确告知用户提取失败，并建议用户直接点击链接查看。

 * 目的：提高用户体验的鲁棒性，避免提供错误信息。

2. 规则二：处理多类别/多页面浏览
 * 当需要访问多个不同的分类或链接时，必须遵循“先打开链接，再截图/快照”的顺序。即：对 URL A 执行 `open` -> 对 URL A 执行 `snapshot`，然后再处理 URL B。
 * 目的：防止上下文污染。如果在同一个浏览器上下文中连续打开多个页面而不及时处理，程序可能会错误地将页面 A 的内容归因于页面 B，或者导致内存/状态混乱。

 3. 规则三：保留原始链接
 * 在向用户返回结果时，允许（并建议）附上原始的 URL 链接。方便用户查看原文；
 * 目的：作为兜底方案，如果程序提取的内容不完整或不准确，用户可以通过原始链接自行验证。

## 实现原理

1.  DOM 解析与容错：程序（如浏览器自动化工具 Selenium、Playwright 或基于 LLM 的浏览器工具）会解析网页的 HTML 树。规则一要求程序在解析逻辑中包含 `try-catch` 机制或特定的验证逻辑。当选择器找不到元素时，触发异常处理分支，输出预设的提示语。
2.  原子性操作：规则二强调操作的原子性。在多任务调度中，确保任务 A（打开 URL + 截图）完全结束后，才重置状态并开始任务 B。这通常通过维护一个任务队列或状态机来实现，确保浏览器上下文在每次新任务开始前是隔离的或已重置的。
3.  上下文注入：规则三涉及到输出格式化。程序在生成最终回复时，不仅提取处理后的文本数据，还会将输入参数中的 URL 变量一并拼接到输出字符串中。
